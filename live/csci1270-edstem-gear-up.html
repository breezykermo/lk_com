What is EdStem?

Semi-standardized discussion forum for Brown CS classes.

The first entry point for students to get asynchronous help.

EdStem | TA hours

Asynchronous Synchronous
Problem visible to all TAs Problem visible to one TA
Optionally public (visible to all students) Exclusively private (visible to one student)
Problem can be solved in hours, days Problem must be solved within 10/20 minutes

Edstem first

How to encourage students to post to EdStem:

  • Encourage students to post privately on EdStem with specific problems.
  • As TA staff, we can then decide to make a post public if we think it will be useful.
  • Create a clear protocol for linking to student code in EdStem posts.
  • Show status of EdStem posts so that students know their problem is being considered.
  • Respond as promptly and comprehensively to questions as possible.

Encouraging EdStem

Suggestion: we explicitly encourage to post on EdStem before coming to TA hours, and to provide a link to the EdStem post in their TA hour sign-up.

This will discourage arbitrary TA hour sign-ups, and will allow preempting of solutions in some cases (if a TA is able to solve the problem on EdStem before the TA hour slot).

Linking to student code

Encourage students to do this using Github, which allows linking to blocks and specific lines of code.

Example:

I am unable to get the streaming stencil running. Running Windows 10, there seems to be an issue with the notebook container. (I think it relates to this code in the docker-compose.yml file.)

EdStem new question triaging workflow

First phase

  • Is it a question for which we already have an answer?
  • Does the question need more context from the student?
  • Has the student assigned the right categories to the question (i.e. “Assignment 1”); should we add any categories?
  • Should the question be looked at more comprehensively by a TA?

Second phase

  • If the question should be looked at, assign it to a TA, and reply to the student saying “Assigned to a TA.”

Third phase

  • Respond. Assigned TA becomes responsible.
  • Track progress on EdStem posts that have been opened for a long time.
  • Institute good knowledge sharing between TAs when struggling to help.

Automation

Eventually we will aim to automate this process. To begin with, Lachlan will do all first passes, and will (likely) create a Google Sheet to track status.

As we get more familiar with EdStem as a system, we will hopefully improve this protocol.

Responding as promptly as possible

  • Create a TA ’playbook’ for responding ot certain kinds of questions, for example on particular assignments.
  • Manage our own backlog of EdStem requests effectively. (During downtime in TA hours, for example.)
  • Suggestion: create an ’on call’ TA schedule for first pass triaging of EdStem. (This will likely happen in Week 1 or 2, once Lachlan has set up the system.)
